[[assignments]]
= Assignments

_Assignments_ are statements that modify the state of a Perspectives model at runtime. They are used in <<state_declarations-automatic-effects,automatic effects>> (`do` blocks), in <<action-declaration,actions>>, and in `letA` blocks.

Every assignment modifies contexts, roles, or property values. Assignments may be preceded by a `letA` block that binds intermediate expression results to local variables.

[[assignments-let]]
== letA — local variable bindings

[source]
----
letA
  <name> <- <expression>
  ...
in
  <assignment>*
----

The `letA` construct binds query expressions to lowercase variable names. Variables are evaluated once and may be reused in subsequent bindings or in the assignment statements that follow.

Example:
[source, arc]
----
letA
  app <- create context MyApp
  start <- create role StartContexts in sys:MySystem
in
  bind_ app >> extern to start
  Name = "My App" for start
----

NOTE: Use `letA` (not `letE`) when you want assignment statements in the body. `letE` is for expression-only pipelines.

[[assignments-property]]
== Property assignments

=== Set a property value

[source]
----
<PropertyName> = <expression> [for <roleExpression>]
----

Replaces the current value(s) of `<PropertyName>` with the result of `<expression>`. The optional `for <roleExpression>` clause specifies the role instance on which the property is set; when omitted, the default role instance determined by the enclosing context is used.

Example:
[source, arc]
----
Status = "active"
Title = "My Document" for extern
----

=== Add a property value

[source]
----
<PropertyName> =+ <expression> [for <roleExpression>]
----

Adds the result of `<expression>` to a relational (multi-valued) property. When the property is functional (single-valued), `=+` behaves identically to `=`.

Example:
[source, arc]
----
Tags =+ "urgent" for origin
----

=== Remove a property value

[source]
----
<PropertyName> =- <expression> [for <roleExpression>]
----

Removes the specific value produced by `<expression>` from a relational property.

Example:
[source, arc]
----
Tags =- "draft" for origin
----

=== Delete a property

[source]
----
delete property <PropertyName> [from <roleExpression>]
----

Removes _all_ values of `<PropertyName>` from the role instance. The optional `from` clause identifies the role instance; when omitted, the default role instance is used.

Example:
[source, arc]
----
delete property DraftNote
delete property CacheValue from origin
----

[[assignments-role]]
== Role assignments

=== Create a role instance

[source]
----
create role <RoleName> [named <expression>] [in <contextExpression>]
----

Creates a new instance of `<RoleName>` in the current context (or in `<contextExpression>` when given). The optional `named <expression>` clause provides a local name for the new role instance, useful when the role requires an indexed name.

Example:
[source, arc]
----
create role Participant
create role Task named ("task-" + nextId) in Projects
----

=== Remove a role instance

[source]
----
remove role <expression>
----

Removes the role instance(s) produced by `<expression>` from their context. The underlying resource is _not_ deleted; use `delete role` for permanent removal.

Example:
[source, arc]
----
remove role origin
remove role filter Tasks with Cancelled
----

=== Delete a role instance

[source]
----
delete role <RoleName> [from <contextExpression>]
----

Permanently deletes the role instance of `<RoleName>` (and the underlying resource). The optional `from` clause identifies the context; when omitted, the current context is used.

Example:
[source, arc]
----
delete role TempData
----

=== Bind (fill) a role

[source]
----
bind <expression> to <RoleName> [in <contextExpression>]
----

Fills an existing, unfilled instance of `<RoleName>` with the role instance(s) produced by `<expression>`. If no unfilled instance exists, one is created first.

Example:
[source, arc]
----
bind me to Participant
bind currentactor to Manager in Projects
----

=== Bind to an explicit binder (`bind_`)

[source]
----
bind_ <expression> to <binderExpression>
----

Fills the binder role instance produced by `<binderExpression>` with the role instance produced by `<expression>`. The binder must be an existing role instance.

Example:
[source, arc]
----
bind_ app >> extern to start
----

=== Unbind a role

[source]
----
unbind <expression> [from <RoleName>]
----

Detaches the filler of the role instance(s) produced by `<expression>`. The optional `from <RoleName>` clause restricts removal to fillers of a specific named binder role.

Example:
[source, arc]
----
unbind me from Participant
unbind currentactor
----

=== Unbind from a specific binder (`unbind_`)

[source]
----
unbind_ <expression> from <binderExpression>
----

Detaches the role produced by `<expression>` from the specific binder instance produced by `<binderExpression>`.

Example:
[source, arc]
----
unbind_ origin from binder SomeRole
----

=== Move a role

[source]
----
move <expression> [to <contextExpression>]
----

Moves the role instance(s) produced by `<expression>` to the context produced by `<contextExpression>`. When `to` is omitted, the role is moved to the current context.

Example:
[source, arc]
----
move origin to TargetContext
----

[[assignments-context]]
== Context assignments

=== Create a context

[source]
----
create context <ContextType> [named <expression>] [bound to <RoleType>] [in <contextExpression>]
----

Creates a new instance of `<ContextType>`. Options:

* `named <expression>` — provides a local name or indexed name for the new context.
* `bound to <RoleType>` — fills the named role type in the target context with the new context's external role.
* `in <contextExpression>` — places the context role into the specified context; when omitted, the current context is used.

Example:
[source, arc]
----
create context Project
create context Meeting bound to Meetings in sys:MySystem
----

=== Create a context with an explicit filler (`create_`)

[source]
----
create_ context <ContextType> [named <expression>] bound to <roleExpression>
----

Creates a new instance of `<ContextType>` and fills the role instance produced by `<roleExpression>` with its external role. Unlike `create context`, this form takes a role _expression_ rather than a role _type name_.

Example:
[source, arc]
----
create_ context Meeting bound to contextRole
----

=== Remove a context

[source]
----
remove context <expression>
----

Removes the context role instance(s) produced by `<expression>` from their parent context. The context itself is _not_ deleted.

=== Delete a context

[source]
----
delete context bound to <ContextRoleName> [from <contextExpression>]
----

Permanently deletes the context that is bound (via its external role) to `<ContextRoleName>`. The optional `from` clause identifies the containing context.

Example:
[source, arc]
----
delete context bound to Projects
----

[[assignments-file]]
== File creation

[source]
----
create file <fileNameExpression> as "<mimeType>" in <PropertyName> [for <roleExpression>]
  <contentExpression>
----

Creates a file with:

* a name given by `<fileNameExpression>` (a string expression),
* a MIME type given by the literal string `<mimeType>` (e.g. `"text/plain"`, `"text/json"`),
* stored as the value of `<PropertyName>` (which must be a `File` property),
* its content provided by `<contentExpression>` (indented on the next line).

Example:
[source, arc]
----
create file ("export_" + context >> Title + ".json") as "text/json" in MyExport
  callExternal ser:SerialiseFor( [role SomeRole], extern ) returns String
----

[[assignments-effects]]
== External effects

=== `callEffect`

[source]
----
callEffect <EffectName>(<arg> {, <arg>}*)
----

Invokes a _non-destructive_ external effect function by name. The function may read data but must not modify the model state.

=== `callDestructiveEffect`

[source]
----
callDestructiveEffect <EffectName>(<arg> {, <arg>}*)
----

Invokes an _destructive_ external effect function that may modify data outside the Perspectives model (for example, sending an email, writing to an external database).

Example:
[source, arc]
----
callDestructiveEffect notifications:SendEmail( EmailAddress, Subject, Body )
----
